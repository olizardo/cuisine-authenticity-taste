# Plan - Create Structured Codebook in Quarto Markdown

This plan outlines the steps to create a comprehensive, highly-structured codebook as a Quarto document (`codebook.qmd`) based on the survey questionnaire (`Final_Survey_Text.txt`), the dataset (`cultdat.dta`), its variable metadata (`variables_metadata.txt`), and the recoding logic (`recode.dat.R`).

## Objectives
1. **Information Synthesis**: Merge information from the survey questionnaire text, Stata variable labels, and R recoding code into a single source of truth.
2. **Quarto Codebook Generation**: Create `codebook.qmd` containing structured sections, variable-to-question mappings, value labels, and recoding notes.
3. **Accessibility**: Provide clear loading instructions in R (via `haven`) and Python (via `pandas`), as well as a comprehensive lookup table for researchers.

## Proposed Codebook Structure

The `codebook.qmd` will contain the following detailed sections:

1. **Metadata & Overview**:
   - Description of the dataset `cultdat.dta` and its role in the study on cuisine authenticity and taste.
   - Quick instructions for loading and preprocessing the data in R and Python.
2. **Global Variables Directory Table**:
   - A complete, searchable Markdown table mapping:
     - Survey Block / Question Number
     - Raw Stata Variable Name
     - Recoded Variable Name (if modified in `recode.dat.R`)
     - Description / Concept
     - Measurement Scale & Value Labels
3. **Detailed Block-by-Block Documentation**:
   - For each survey block, document the original question text, the mapped Stata variable(s), the response options, and any R-recode transformations (factors, categories collapsed, etc.):
     - *Consent & Attention Checks* (Consent Q1-Q4, Carrot Juice check)
     - *Art Judgments (IsItArt)* (`crayon`, `paw_patrol`, `rorshach`, `last_supper`)
     - *Social Distinctions (KnowAPerson)* (`knowaperson1` to `knowaperson14`)
     - *Aesthetic Judgments (Photo)* (`landscape` to `snake` - 20 items, beautifully listed)
     - *Self-Presentation (Fashion)* (`fashion1` to `fashion10`)
     - *Cultural Practices (Practice)* (21 activities including `fancy_restaurant`, `fast_food`, etc.)
     - *Bourdieu Food* (Cuisine preferences: `light`, `rich`, `authentic`, `familiar`, `big`, `small`, `caviar`, `oysters`, etc.)
     - *Foodies (Cuisine Authenticity)* (15 cuisines: `japanese` to `lebanese` on traditional elder vs high-end chef scale, and their factor recodings `japanese.f` to `lebanese.f`)
     - *Attitudes & Ideology (Fields)* (`uniquetastes`, `bestartpolitical`, `bestartmoral`, `bestartnotpolitical`, `bestartnotmoral`, `cancelledartists`, `polarized`, `enjoyabledifficult`, `ironic`, `comedy` - and their renamed forms)
     - *Political Animus* (Democrats/Republicans characteristics: Intelligent, Open-minded, Generous, Hypocritical, Selfish, Mean)
     - *Identity & Cosmopolitanism (cosmo)* (Neighborhood, State, Nation, Continent, Globe influence)
     - *Social Media Usage* (Facebook, Instagram, etc. - hours per week)
     - *Social Networks (Weak & Strong Ties)* (Network diversity and size)
     - *Demographics* (Gender, Sexual orientation, Birth year/Age, Race, School status, Education, Income, Urban/Rural residency)

## Proposed Steps

### Step 1: Draft the Quarto Codebook (`codebook.qmd`)
- We will write a beautifully formatted `codebook.qmd` document with appropriate YAML metadata, styled Callout boxes, and custom formatting for lists and tables to ensure high readability.
- We will integrate the raw survey text with the metadata from `variables_metadata.txt` and the exact recodes from `recode.dat.R`.

### Step 2: Validate Document Alignment
- Verify that every variable referenced in `recode.dat.R` and `variables_metadata.txt` is documented with its corresponding survey question and scale options.
- Double-check the exact mapping of factor levels (e.g., how the 15 cuisines are collapsed from a 1-7 scale into `"Trad"`, `"Not Sure"`, and `"High-End"`).

### Step 3: Compile and Render (Optional)
- Offer to render the codebook to HTML (`codebook.html`) or PDF using Quarto so the user has an instantly readable compiled output.
