# Plan - Code Audit, Verification, and Report Compilation

This plan outlines the steps to audit the analysis code in `recode.dat.R` and `main-analysis.qmd` for correctness, efficiency, and accuracy, run all analyses, and compile/update the `statistical-findings-report.qmd` and its compiled HTML format.

## Objectives
1. **Code Audit & Optimization**:
   - Audit `recode.dat.R` for efficiency issues (such as `rowwise()` operations) and robustness issues (such as `factor()` labeling without explicit `levels`).
   - Verify that column groupings in `MFA()` exactly match the variable selection counts to prevent group size mismatches.
   - Audit `main-analysis.qmd` to ensure all model definitions, plotting functions, and file-saving operations are correct, robust, and use current best practices (e.g., proper theme formatting, avoidance of deprecated arguments).
2. **Analysis Execution**:
   - Run the updated and audited analysis scripts sequentially or via chunk execution to verify zero-error runs.
   - Ensure all output files (plots in `Plots/` and tables in `Tabs/`) are generated correctly and reflect the audited, validated code.
3. **Report Verification & Compilation**:
   - Verify that `statistical-findings-report.qmd` aligns perfectly with the audited analysis.
   - Render the Quarto report to HTML (`statistical-findings-report.html`) and check for any build errors, warnings, or missing elements.

## Proposed Steps

### Step 1: Audit and Optimize Recoding Code (`recode.dat.R`)
- **Robust Factor Levels**: In `recode.dat.R`, several variables are converted to factors with specific labels (e.g., `landscape:snake` using `labels = c("Be", "In", "Me", "Ug")`). If any variable is missing a level in the raw data, this will lead to incorrect mapping. We will update these to explicitly specify `levels = 1:4` alongside `labels` to ensure robust, accurate categorization.
- **Performance Optimization**: The rowwise variance of `landscape:snake` is computed using `rowwise() |> mutate(var.photo = var(c_across(landscape:snake)))`. We will optimize this slow rowwise operation using a faster, vectorized alternative like row-wise variance via `apply()` or `matrixStats`.
- **Reference Levels Verification**: Double-check that all `relevel()` commands are correct and correspond to existing factor levels in the recoded data.

### Step 2: Audit and Verify Main Analysis Code (`main-analysis.qmd`)
- **Group Size Verification**: Programmatically check that the selected columns in `dat` for the Multiple Factor Analysis (MFA) exactly match the group size vector `c(2, 6, 6, 15, 7, 5)` to avoid any dimension mismatch.
- **Plotted Aesthetics**:
   - Review qualitative coordinate plotting and correlation heatmap plotting to ensure proper scaling, legends, and avoidance of warning-generating code.
   - Ensure that the interaction plots handle discrete factors on the x-axis properly without missing groups.
- **Model Check**:
   - Validate that `summary(manova_model, test = "Pillai")` and the subsequent table generation are robust to changes in variable ordering or missing values.
   - Ensure `tab_model` writes clean HTML outputs.

### Step 3: Run the Analyses & Generate Outputs
- Run `main-analysis.qmd`'s chunks or script.
- Verify that the plots (`Plots/eig.png`, `Plots/cat_coords.png`, `Plots/varcor.png`, `Plots/interaction_dim1.png`, `Plots/interaction_dim2.png`, `Plots/pred_prob_faceted.png`) and tables (`Tabs/manova_table.html`, `Tabs/pairwise_manovas.html`, `Tabs/tukey_hsd_dim1.html`, `Tabs/multinom_table.html`) are saved without errors.

### Step 4: Update and Render the Report (`statistical-findings-report.qmd`)
- Align the code chunks in `statistical-findings-report.qmd` with the optimizations and corrections made to `main-analysis.qmd` and `recode.dat.R`.
- Render the report to HTML using Quarto (`quarto render statistical-findings-report.qmd`).
- Review the generated HTML document to ensure it renders correctly, displaying all tables and figures.
